# Initial Concept

Веб-застосунок на render на безкоштовному таріфі. Це застосунок для моєї рольової гри, де буде: менеджер сесій, термінал ведучого та термінали всіх гравців.

---

# Product Definition: Role-playing Game Assistant (RPG-Web-Term)

## Vision
Веб-застосунок на render на безкоштовному таріфі для рольових ігор. Це інструмент для керування сесіями, який забезпечує взаємодію ведучого та гравців через термінальний інтерфейс у реальному часі.

## Target Audience
- **Ведучий (GM):** Головний адміністратор сесії, який керує світом, ініціативами та логами.
- **Гравці:** Учасники, які взаємодіють зі світом через власні термінали та керують своїми листами персонажів.

## Core Goals
1.  **Полегшення гри (Streamlining):** Автоматизація рутинних обчислень, логування подій та кидків кубиків.
2.  **Централізоване управління (Hub):** Всі дані про сесію, стан персонажів та історію гри зберігаються в одному місці.
3.  **Взаємодія в реальному часі (Sync):** Миттєва синхронізація дій GM та оновлень у терміналах гравців.

## Key Features (MVP)
- **Менеджер сесій (Session Manager):** Створення нових ігор, завантаження існуючих та керування статусом сесії (активна/пауза).
- **Термінальний інтерфейс (Terminal UI):** Текстовий ввід команд для гравців та GM, відображення результатів дій та логів гри.
- **Листи персонажів (Character Sheets):** Цифрові профілі гравців з основними характеристиками, які динамічно оновлюються.

## Technical Constraints & Principles
- **Render Free Tier Optimization:** Врахування особливостей безкоштовного тарифу (затримка при першому завантаженні, обмеження ресурсів).
- **Lightweight Stack:** Використання мінімально необхідних ресурсів та ефективних рішень для збереження стану без важких БД.
- **Real-time Sync (Lite):** Забезпечення синхронізації через WebSockets або Server-Sent Events (SSE) для швидкого відгуку терміналів.
